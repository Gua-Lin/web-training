- node_modules：项目依赖包，存第三方库，不用手动改
- public：放静态资源
- src：核心代码区，主要改App.css和App.tsx

- devDependencies：开发的依赖

- 能解释：
  
  - 为什么不用 document.querySelector？
  - document.querySelector 是原生 JS 手动根据 id 找标签用的。
  - React 是数据驱动，只需要改 state 数据，页面就会自动变化，不用手动找标签操作 DOM。
  
  - React 和原生 JS 的区别？
  - 原生 JS 要先根据 id 找到标签，才能做操作；React 不需要找标签，直接改数据就能控制页面，配置更少、响应更快。

## 功能

  - 保存 token

- 失败：
  
  - 页面提示（不能只 alert）

启动后端服务

```bash
node server.js
```

server.js代码示例

```javascript
const http = require("http");

const server = http.createServer((req, res) => {
  // 只处理 POST /login
  if (req.method === "POST" && req.url === "/login") {
    let body = "";

    // 接收数据
    req.on("data", chunk => {
      body += chunk;
    });

    // 数据接收完毕
    req.on("end", () => {
      try {
        const data = JSON.parse(body);
        const { username, password } = data;

        if (username === "admin" && password === "123456") {
          res.writeHead(200, {
            "Content-Type": "application/json"
          });
          res.end(JSON.stringify({
            code: 0,
            data: {
              token: "node-token-123"
            }
          }));
        } else {
          res.writeHead(401, {
            "Content-Type": "application/json"
          });
          res.end(JSON.stringify({
            code: 1,
            message: "用户名或密码错误"
          }));
        }
      } catch (err) {
        res.writeHead(400);
        res.end("Bad Request");
      }
    });
  } else {
    res.writeHead(404);
    res.end("Not Found");
  }
});

server.listen(3000, () => {
  console.log("Server running at http://localhost:3000");
});
```

---

## 📚 必学

- Promise / async-await

- 请求 vs 响应

- try/catch

---

## ✅ 验收点

- 能说清：
  
  - 请求是怎么发出去的？
  - 点击登录，触发点击事件；校验非空之后，调用axios的post方法，向后端发送用户名跟密码；请求被vite代理转发给后端接收校验，返回成功/失败；成功则存储token，失败则提示
  
  - 为什么要 async/await？
  - axios发送网络请求是异步操作，async/await是其中的语法糖，

- 页面有 loading状态
- 登录中的状态，在点击事件发生时loading状态true，请求结束时在 finally 中把 Loading 设为 false

# 🧩 Day 5：登录态 + 路由控制

## 🎯 任务

实现页面跳转 + 登录保护

---

## 技术要求

使用：  
👉 React Router

---

## 功能

- 登录成功 → `/home`

- 未登录访问 `/home` → 跳 `/login`

---

## 📚 必学

- SPA
- 单页面应用，一个html不用完全刷新，通过js动态切换的组件就可以页面不刷新，组件变化就行

- 路由原理
- 监听URL变化，匹配对应组件来渲染。

- localStorage
- token存到本地就可以持久登录，不用重复登录。
- localStorage：数据永久存储，除非手动删除或清除浏览器数据

---

## ✅ 验收点

- 刷新页面仍保持登录状态
- 登录之后将token存在localStorage里面，localStorage一般不会清理，所以可以保持登录状态。

- 能解释：
  
  - token 为什么存在？
  - 存在localStorage里面，可以校验是否登陆成功；在请求时作为凭证，可获取数据。
  
  - 为什么前端能控制跳转？
  - 是SPA；路由接管了浏览器的url管理；可以根据条件来决定跳转到哪个页面；速度快，不需要请求后端。
